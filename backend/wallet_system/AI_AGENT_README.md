# AI Agent المدمج داخل مشروع Wallet System

هذه النسخة تجمع بين:

- Agent الداخلي في المشروع
- منطق `wallet.py`

لكن **بدون أي مفاتيح API أو توكنات ثابتة داخل الكود**، وبدون استدعاء LLM خارجي.

يعتمد الوكيل هنا على:

- المستخدم الحالي المسجل دخوله عبر Bearer token الطبيعي للنظام
- خدمات المشروع الداخلية مباشرة
- تحليل نصي محسن لاستخراج القيم من الرسالة
- استنتاج تلقائي للحساب حسب العملة
- استنتاج تلقائي للبنك في طلب السحب من الصراف

## الملفات المهمة

- `apps/agent/schemas.py`
- `apps/agent/services.py`
- `apps/agent/routes.py`
- `main.py`

## المسارات الجديدة

### 1) جلب قدرات الوكيل

`GET /agent/capabilities`

### 2) تنفيذ طلب ذكي

`POST /agent/execute`

## العمليات المدعومة الآن

- `GET_WALLET_SUMMARY`
- `LIST_TRANSACTIONS`
- `LIST_FAILED_TRANSACTIONS`
- `TRANSFER_MONEY`
- `MOBILE_TOPUP`
- `EXCHANGE_MONEY`
- `ATM_WITHDRAW_REQUEST`
- `GET_EXCHANGE_RATE`
- `LIST_CURRENCIES`
- `ADD_FAVORITE_CONTACT`

## ما الذي تم دمجه من wallet.py ؟

تم نقل أهم نقاط القوة من `wallet.py` إلى داخل agent المشروع نفسه:

- دعم طلب السحب من الصراف
- دعم عرض العمليات الفاشلة
- محاولة تحديد حساب المستخدم تلقائياً حسب العملة
- محاولة تحديد حسابي المصدر والهدف تلقائياً في الصرف
- محاولة تحديد البنك تلقائياً بالاسم أو الكود أو bank_id
- تلخيص أفضل للعمليات

## ما الذي تم حذفه؟

تم حذف أي اعتماد على:

- `GROQ_API_KEY`
- `WALLET_ACCESS_TOKEN` الثابت داخل الملف
- أي استدعاء HTTP خارجي من وكيل منفصل

## أمثلة

### مثال عرض الرصيد

```json
{
  "message": "اعرض رصيدي"
}
```

### مثال تحويل تلقائي حسب العملة

```json
{
  "message": "حول 100 دولار إلى 7750514050"
}
```

### مثال شحن رصيد

```json
{
  "message": "اشحن 5000 للرقم 777123456"
}
```

### مثال صرف بين العملات

```json
{
  "message": "اصرف 50 من الدولار إلى الريال"
}
```

### مثال طلب سحب من الصراف

```json
{
  "message": "اريد سحب 20000 ريال من الصراف بنك الكريمي"
}
```

### مثال العمليات الفاشلة

```json
{
  "message": "اعرض آخر 10 عمليات فاشلة"
}
```

## ملاحظات مهمة

- ما زال الوكيل يحتاج **تسجيل دخول طبيعي** للنظام عند الاستدعاء.
- لا توجد أي مفاتيح أو أسرار مضمنة داخل الكود.
- العمليات المالية تتطلب أن يكون المستخدم **موثق KYC**.
- هذه النسخة أقوى من النسخة السابقة لأنها جمعت وظائف الطرفين مع الحفاظ على بنية المشروع الأصلية.
